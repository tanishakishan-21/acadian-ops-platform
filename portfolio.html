The Problem

What's Broken in Investment Operations

Acadian manages $178B across 40,000+ securities globally. Based on the Investment Operations role requirements and industry research, these are the four pain points this platform addresses.

Industry Fact: Asset managers spend a significant amount of time gathering, cleaning, and normalizing data before it can be used. Relying on spreadsheets and emails is no longer enough. — BNY Future of Asset Management Report, 2024
📊

No Centralized Visibility

Portfolio managers lack a single view of daily positions health across all strategies — Global Equity, Low Vol, ESG, EM, and Multi-Asset.

⚠️

Reactive Data Quality

Missing prices, unverified positions, and reconciliation breaks are discovered late, often after they've impacted downstream processes.

💸

Settlement Risk

Failed and pending settlements are tracked in disparate systems. Duplicate trades go undetected until end-of-day reconciliation.

📄

Static Reporting

Management receives PDF reports instead of live dashboards — slowing decision-making in a firm that generates expected returns multiple times per day.

SQL Queries

11 Analytical Queries

Written in PostgreSQL. Each query powers a specific section of the dashboard or supports a core BA responsibility — data mapping, reconciliation, and incident detection.

Query 6 — Duplicate Transaction Detection (Critical Data Quality Check) 
-- Automatically flags potential duplicate trades before settlement
SELECT
    t1.portfolio_id,
    sec.ticker,
    t1.transaction_type,
    t1.trade_date,
    t1.quantity,
    t1.price,
    t1.broker,
    COUNT(*) AS duplicate_count,
    STRING_AGG(t1.transaction_id, ', ') AS transaction_ids
FROM transactions t1
JOIN securities sec ON t1.security_id = sec.security_id
GROUP BY
    t1.portfolio_id, t1.security_id, sec.ticker,
    t1.transaction_type, t1.trade_date,
    t1.quantity, t1.price, t1.broker
HAVING COUNT(*) > 1
ORDER BY duplicate_count DESC;
Query 10 — Position-Transaction Reconciliation (Core BA Responsibility) 
-- Validates that transaction activity matches position changes between two dates
WITH position_changes AS (
    SELECT p1.portfolio_id, p1.security_id,
           p1.quantity AS qty_day1,
           p2.quantity AS qty_day2,
           (p2.quantity - p1.quantity) AS position_change
    FROM positions p1
    JOIN positions p2
        ON  p1.portfolio_id = p2.portfolio_id
        AND p1.security_id  = p2.security_id
        AND p1.position_date = '2026-02-10'
        AND p2.position_date = '2026-02-11'
),
transaction_activity AS (
    SELECT portfolio_id, security_id,
           SUM(CASE WHEN transaction_type = 'BUY'  THEN quantity ELSE 0 END)
         - SUM(CASE WHEN transaction_type = 'SELL' THEN quantity ELSE 0 END)
           AS net_txn_qty
    FROM transactions
    WHERE trade_date BETWEEN '2026-02-10' AND '2026-02-11'
    GROUP BY portfolio_id, security_id
)
SELECT
    pc.portfolio_id, sec.ticker,
    pc.position_change,
    COALESCE(ta.net_txn_qty, 0) AS net_transaction_qty,
    (pc.position_change - COALESCE(ta.net_txn_qty, 0)) AS reconciliation_break,
    CASE WHEN (pc.position_change - COALESCE(ta.net_txn_qty, 0)) = 0
         THEN 'RECONCILED' ELSE 'BREAK - INVESTIGATE'
    END AS reconciliation_status
FROM position_changes pc
LEFT JOIN transaction_activity ta
    ON  pc.portfolio_id = ta.portfolio_id
    AND pc.security_id  = ta.security_id
JOIN securities sec ON pc.security_id = sec.security_id
ORDER BY reconciliation_status DESC;
Business Analysis

User Stories & Acceptance Criteria

Written following Agile BA methodology. Each story maps directly to a dashboard panel and includes testable acceptance criteria.

Investment Operations Manager

Portfolio Health Dashboard

"I want a real-time view of all portfolio positions so I can identify issues before market open."

Acceptance Criteria

✓Color-coded verification status: Green >95%, Yellow 85-95%, Red <85%
✓Missing price count visible at portfolio and security level
✓Asset class exposure shown as % breakdown per portfolio
✓Data refreshes within 15 minutes of market open
Trade Operations Analyst

Transaction Monitor

"I want a live view of pending and failed settlements so I can act before the window closes."

Acceptance Criteria

✓FAILED transactions highlighted in red with broker name and reason
✓Duplicate transactions auto-flagged by broker/quantity/date matching
✓Settlement aging buckets: 0-1 days, 2-3 days, 4+ days
✓All at-risk settlements identifiable within 2 minutes
Business Architect / IT Manager

Data Quality KPI Tracker

"I want a 30-day view of data incidents so I can report operational risk to senior management."

Acceptance Criteria

✓KPI scorecard: open, in-progress, resolved, resolution rate %
✓Issues by type: Missing Price, Duplicate, Settlement Fail, Recon Break
✓30-day trend line: issues raised vs. resolved
✓Management summary pullable in under 60 seconds
