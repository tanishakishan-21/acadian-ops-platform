Problem Statement

What breaks in investment operations

📊

No unified portfolio view

Portfolio health, P&L, and exposure data live in separate systems. Operations teams spend hours reconciling instead of monitoring.

⚠️

Settlement failures go undetected

Failed and pending settlements pile up without a real-time monitor. By the time someone notices, the cost and effort to fix it has multiplied.

🔍

Data quality is invisible

Missing prices, duplicate records, and stale data corrupt downstream analytics. Without a systematic flag, these issues compound silently.

Technical Work

The SQL underneath

portfolio_exposure.sql Window Function 
-- Asset class exposure % per portfolio
SELECT
  p.portfolio_name,
  pos.asset_class,
  SUM(pos.market_value_usd) AS class_value,
  ROUND(
    SUM(pos.market_value_usd) * 100.0
    / SUM(SUM(pos.market_value_usd))
      OVER (PARTITION BY p.portfolio_id), 2
  ) AS exposure_pct
FROM portfolios p
JOIN positions pos
  ON p.portfolio_id = pos.portfolio_id
GROUP BY 1, 2
ORDER BY 1, exposure_pct DESC;
settlement_risk.sql Risk Flag 
-- Flag pending settlements past T+2
SELECT
  t.transaction_id,
  p.portfolio_name,
  t.ticker,
  t.settlement_status,
  t.trade_date,
  t.settlement_date,
  CURRENT_DATE - t.settlement_date
    AS days_overdue
FROM transactions t
JOIN portfolios p
  ON t.portfolio_id = p.portfolio_id
WHERE
  t.settlement_status IN (
    'PENDING', 'FAILED'
  )
  AND CURRENT_DATE > t.settlement_date
ORDER BY days_overdue DESC;
data_quality_score.sql Aggregation 
-- DQ score by portfolio and severity
SELECT
  portfolio_name,
  COUNT(*) AS total_issues,
  SUM(CASE WHEN severity = 'HIGH'
    THEN 1 ELSE 0 END) AS high_sev,
  ROUND(
    SUM(CASE WHEN
      resolution_status = 'RESOLVED'
      THEN 1.0 ELSE 0 END)
    / COUNT(*) * 100, 1
  ) AS resolution_rate_pct
FROM data_quality_log
GROUP BY 1
ORDER BY high_sev DESC;
pnl_summary.sql Ranking 
-- Top performing positions by P&L
SELECT
  p.portfolio_name,
  pos.ticker,
  pos.asset_class,
  pos.unrealized_pnl,
  RANK() OVER (
    PARTITION BY pos.portfolio_id
    ORDER BY pos.unrealized_pnl DESC
  ) AS pnl_rank
FROM positions pos
JOIN portfolios p
  ON pos.portfolio_id = p.portfolio_id
WHERE pos.unrealized_pnl > 0
ORDER BY p.portfolio_name, pnl_rank;
